# SP-404 Pad Sampler (`sp404`)

## Overview

A 16-pad sample player inspired by the Roland SP-404. Each pad holds one sample, triggered by MIDI note or gate input. Designed for beat-making, sound design triggers, and live performance.

## Parameters

| Name | Type | Default | Range | Description |
|------|------|---------|-------|-------------|
| `file` | FilePath | — | — | Path to a sample pack JSON file |
| `mode` | enum | one_shot | one_shot / loop / gate | Playback behavior per pad |
| `attack` | float | 0.001 | 0.001–2.0 | ADSR attack time (seconds) |
| `decay` | float | 0.1 | 0.01–2.0 | ADSR decay time |
| `sustain` | float | 1.0 | 0.0–1.0 | ADSR sustain level |
| `release` | float | 0.05 | 0.001–10.0 | ADSR release time |
| `volume` | float | 1.0 | 0.0–2.0 | Master output volume |

## Ports

### Inputs (spread)
- `notes` — MIDI note numbers (from MidiInput operator)
- `velocities` — MIDI velocities (0.0–1.0)
- `gates` — gate signals (>0.5 = on)

### Outputs (audio)
- `left` — left channel audio
- `right` — right channel audio

## Sample Pack JSON Format

The SP-404 uses the same JSON format as the other sampler operators, but each sample maps to exactly one note (pad):

```json
{
  "name": "My Kit",
  "samples": [
    { "path": "kick.wav",  "root_note": 36, "lo_note": 36, "hi_note": 36 },
    { "path": "snare.wav", "root_note": 37, "lo_note": 37, "hi_note": 37 },
    { "path": "hat.wav",   "root_note": 38, "lo_note": 38, "hi_note": 38 }
  ],
  "envelope": {
    "attack": 0.0, "decay": 0.0, "sustain": 1.0, "release": 0.05
  }
}
```

Default pad mapping: MIDI notes 36–51 (C2–D#3), matching the General MIDI drum map starting at kick drum.

## Playback Modes

- **one_shot** — sample plays to completion once triggered, regardless of gate state. Re-triggering restarts from the beginning.
- **loop** — sample loops continuously while gate is held. On gate-off, enters ADSR release.
- **gate** — sample plays while gate is held, enters ADSR release on gate-off. Does not loop.

## Voice Allocation

16 voices (one per pad). Each pad can only have one active voice at a time — re-triggering the same pad cuts the previous voice and restarts. Different pads play simultaneously.

No pitch interpolation — samples play at their recorded pitch. Sample rate mismatch is corrected: `playback_rate = sample_sr / runtime_sr`.

## Processing

Per audio buffer (256 samples at 48kHz):
1. Read spread inputs, detect gate edges (rising = note-on, falling = note-off)
2. On note-on: find the sample region matching the note, allocate/restart voice
3. On note-off: if mode is gate or loop, enter ADSR release
4. For each active voice, render samples with ADSR envelope applied
5. Sum all voices into stereo output buffers

## Thread Safety

- Sample loading happens in `main_thread_update()` (not on the audio thread)
- Loaded `SampleBank*` is swapped in via `std::atomic` pointer
- Old bank pointer is deferred-deleted on the next `main_thread_update()` call
- `process_audio()` reads the atomic pointer and never blocks

## Dependencies

- `vivid::adsr` from `operator_api/adsr.h` — envelope generation
- `miniaudio` — WAV file decoding
- `yyjson` — JSON parsing
- Shared infrastructure: `sample_bank.h`, `voice.h`
